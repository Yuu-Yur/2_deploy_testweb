Chapter.10

연결 선택자

보통 css에서 tag를 선택하여 style을 부여하면 모든 tag가 선택이 되기에 class나 id를 부여하여 관리한다

그러나 이러한 방식도 부여하는 작업이 필요하므로 수가 커질수록 반복작업이 많아진다

이럴 때 미리 정해진 몇가지의 방법으로 class나 id를 부여하지 않고도 특정한 tag를 선택하여 관리할 수 있다

  • 하위 선택자 : 가장 간단한 방법으로 특정 부모아래의 모든 자식을 선택한다 ex fieldset ul font size : 18px;
  • 자식 선택자 : 하위 선택자와 비슷하지만 특정 부모를 가진 모든 자식을 선택한다 하위 선택자가 부모가 주라면 자식 선택자는 자식이 주이다 ex div > h2 color violet 이것으로 부모가 div인 모든 h2의 글자색은 violet이 된다
  • 인접 형제 선택자 : 형제 중 가장 첫번째를 선택한다 ex legend + p fontweight bold 이것으로 legend의 형제중 첫번째 p가 bold가 된다
  • 형제 선택자 : 형제 모두를 선택한다 ex legend ~ p fontstyle italic 이것으로 legend의 형제인 p 모두가 italic이 된다
속성 선택자

연결 선택자는 tag를 의존하여 tag를 선택한다면 속성 선택자는 속성을 의존하여 tag를 선택한다

예를 위해 이 리스트에는 모두 att숫자 의 title을 붙였다

  • 속성 선택자 []안에 원하는 속성을 넣어 선택할 수 있다 []앞에 태그를 지정할 수도 있다 ex li[title] color : crimson;
  • 속성 안의 값도 지정하여 선택할 수 있다 일치는 []안에 속성과 값을 =로 붙인다 ex [title=att1] fontweight : bold;
  • 속성 안에 여러 값이 있을 때에도 특정 속성과 일치하는 값이 있다면 선택할 수 있다 ex [class~=a] fontsize :100px;
  • 일치와 비슷하지만 -이 붙어있는 것 까지 선택하는 특수한 경우에 사용 ex [title|=att4] fontstyle : italic;
  • 특정 값으로 시작하는 속성을 가진 tag 선택 가능 ex [title^=at] background : yellow;
  • 특정 값으로 끝나는 속성을 가진 tag 선택 가능 ex [title$=t6] textalign="center";
  • 일부가 일치 하는 값을 가진 속성의 tag 선택 가능 ex [title*=tt] text-shadow: 2px 1px rebeccapurple;
가상 클래스 선택자와 가상 요소

사용자의 동작에 반응 하는 가상 클래스로서

특정한 상황에 맞추어 스타일을 변경할 수 있다

  • :link 방문한 기록이 없는 링크 ul a:link {color:blue;}
  • :visited 방문한 기록이 있는 링크 ul a:visited {color:red;}
  • :active 요소가 활성화 되었을 때 ex #activetest:active {width: 600px;}
  • :hover 요소 위에 마우스를 올렸을 때 ex #hovertest:hover {width:600px;}
  • :target 링크와 같이 써야 하며 active하면 ex #activetest:target{background : red;}
  • :target 가리키는 대상의 스타일을 변화시킨다 ex #hovertest:target{background : blue;}
  • :enabled, :disabled 해당 대상이 enabled나 disabled일때 스타일을 변화시킨다 ex .abledtest:enabled,disabled {background: red,blue;}
  • :checked 해당 대상이 라디오나 체크 일때 체크했을 경우 스타일을 변화 ex #checkedtest:checked + label {background : black;color : white;}
  • :not 해당 이외의 대상을 선택 ex ul input[type=radio]:not([name=nott2]) + label {background : black}
  • 문서 구조에 따라서 선택도 가능하다 :only-child, :first-child, :last-child, nth-child(n), nth-last_child(n)
  • 임의의 부모 아래에서 임의의 자식이 하나뿐일때 선택, 여러 자식중 첫번째, 여러 자식중 마지막, 여러 자식중 n번째, 여러 자식중 마지막에서 n번째
  • 또 :앞에 tag를 붙인 다음 child를 of type로 바꾸면 임의의 부모를 가진 해당 tag 중에서 각각이 선택된다 예외로 only child만 only type of 로 변환
  • ex #test10 li:last-of-type{font-weight: bold;}

트랜지션

트랜지션은 변형의 과정에 관여하는 스타일이다

시간, 타이밍, 속도, 속성 등을 설정할 수 있다

아래는 같은 변형에 트랜지션을 적용한 것이다

transition: all 1s ease-out 0.3s;

  • translateX translateY translateZ(2차원이라 움직이지 않음)
  • scaleX scaleY scaleZ(2차원이라 움직이지 않음)
  • rotateX rotateY rotateZ rotate3d
  • skewX skewY skew

시작과 끝, 그리고 변화점을 지정하여 animation 속성의 값으로 취급한다

  • @keyframes 은 여러개를 사용할 수 있다, animation: rotate, background
  • 여기에 각 애니메이션의 진행시간이나 반복등을 지정할 수 있다

Chapter.12

반응형 웹

pc의 화면은 물론 계속 다양한 화면 크기의 모바일 기기들이 나오는 중인데

이 화면크기의 차이는 만든 웹 사이트의 스타일을 의도대로 전달하기 어렵게 만든다

이 때 반응형 웹으로 화면 크기에 반응해 요소들을 자동으로 알맞게 바꾸어 일관적인 경험을 전달한다

  • meta name="viewport" content="width=device-width, initial-scale=1.0"
  • viewport를 써서 길이를 장치의 크기에 맞추고 1배율로 보여주는 기본 사용법이다
  • 1vw, 1vh 각각 뷰포트 너비와 높이의 1%를 뜻한다
미디어 쿼리
  • 미디어 쿼리 @media 미디어 유형 and (조건) and 조건 {}
  • ex @media screen and (min-width: 1800px) {}
  • 이것을 사용해서 가로 길이 1800px이상에서 변화를 줄 수 있다
그리드 레이아웃
  • 그리드 레이아웃이란 화면을 박스 형태로 나누어 배치하여 일관성있게 보이게 한다
  • 플렉스 레이아웃과 css 그리드 레이아웃이 있는데 보통 그리드 레이아웃이라 하면 후자를 가리킨다
    • 플렉스 박스 레이아웃
  • 플렉스 박스는 주 축을 기준으로 나열하고 스타일링하며 교차축을 간단하게 조정할 수 있다
  • ex @media screen and (min-width: 1800px) { #flex1800 { display: flex; flex-wrap: wrap; } }
  • 이 페이지에 사용한 예시로 가로 길이 1800px이상에서 행(기본값)을 주축으로 플렉스 배치를 사용하며 여러 행을 사용 할 수 있게 wrap속성을 부여하였다
  • 추가로 justify-content, align-items 를 사용하여 정렬이 가능하며 align-content로 여러 주축을 가질 때도 정렬이 가능하다
    • CSS 그리드 레이아웃
    • 플렉스 레이아웃과 달리 주축 교차축 개념이 없이 표과 비슷하게 행열개념을 사용한다
    • ex #gridShow { display: grid; grid-template-columns: repeat(auto-fit, minmax(320px, 50%)); }
    • 이렇게 가로 길이가 최소 300px, 최대 50%가 되게 자동으로 채울 수도 있고
    • grid-trmplate-columns: 50% 50%로 각각 지정할 수도 있다
    • 그리드 라인 각 행열에는 제일 왼쪽 위부터 번호가 붙는다
    • 이 그리드 같은 경우엔 행이 1,2,3,4 열이 1,2,3
